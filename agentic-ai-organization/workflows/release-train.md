# Release Train

## Cadence
- 2-week development sprint
- 1-week integration hardening
- Monthly platform release
- Quarterly industry-template release

## Release Stages
1. Intake and prioritization
2. Architecture review
3. Story readiness review
4. Module development
5. Module QA
6. Cross-module integration testing
7. Security and compliance review
8. UAT simulation by industry template
9. Release candidate approval
10. Production release
11. Post-release monitoring
12. Feedback backlog update

## Release Gates
- No critical data ownership conflicts
- No unresolved security defects
- Cross-module happy paths pass
- Regression suite passes
- Audit trail verified for critical actions
- Release notes complete
- Rollback plan ready
- Demo scenario ready
