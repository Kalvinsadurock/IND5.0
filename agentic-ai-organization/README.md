# Agentic AI Organization for Industrial Platform Development

This folder defines the AI product-development organization for building a configurable industrial platform that combines MES, ERP, SAP-compatible integration, OEE, HRMS, quality, maintenance, work management, reporting, and industry templates.

## Objective

Build one configurable application that can cover most manufacturing and industrial use cases for companies with roughly 100 to 10,000 employees. The platform must support industry-specific customization without rewriting core code for every customer.

## Operating Principle

Each module is owned by an AI module team. Every team has product, architecture, analysis, development, QA, UX, reporting, and compliance responsibilities. Shared governance keeps data models, UI patterns, APIs, and release quality consistent.

## Primary Module Teams

- Platform Core Team
- Configuration Studio Team
- MES Team
- ERP Team
- SAP Integration Team
- HRMS and Workforce Team
- OEE Team
- Quality and Compliance Team
- Inventory and Warehouse Team
- Maintenance and Asset Team
- Work Management and Ticketing Team
- Reporting, Analytics, and AI Team
- DevOps, Security, and Release Team

## How To Use This Folder

- Start with governance/operating-model.md
- Use agents/agent-roster.md to understand the AI organization
- Use prompts/ to launch specialist agents
- Use templates/ for module charters, user stories, handoffs, and QA gates
- Store module-specific planning in charters/
